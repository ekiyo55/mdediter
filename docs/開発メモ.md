# 開発メモ

開発中に得た知見・決定事項・注意点を随時追記。

## 決定事項

### なぜ Wails？

- Electron（100MB+）ではバイナリが巨大になる
- Tauri は Rust だが、今回は Go の経験を活かしたいため Wails を採用
- Wails v2 は Windows 向けに `wails build` で単一 exe を出せる
- WebView2 は Windows 10 Build 1803+ で標準搭載 → 追加配布不要

### なぜ Svelte？

- Wails のテンプレートが公式で svelte / vue / react を用意
- Svelte はビルドサイズが小さく、runtime overhead が少ない
- 小規模アプリでは記述量も最少

### CodeMirror 6 を選んだ理由

- Monaco は Electron 前提のサイズ感で WebView2 には重い
- CodeMirror 6 はモジュール式で Markdown サポートが公式
- `@codemirror/lang-markdown` で構文木ベースのハイライトが得られる

### markdown-it を選んだ理由

- プラグインエコシステムが豊富
- GFM 対応は `markdown-it-task-lists` などで補完可能
- remark は AST 指向で今回はオーバーキル

## 既知の注意点

- Wails は WebView2 ランタイムが無い Windows では動作しない
  → Windows 10 1803+ / Windows 11 を要件にする
- `wails build` の初回は npm install + go mod download が走り数分かかる
- Windows パスは `\\` エスケープが必要なため、設定ファイルは JSON でなく素のパス保存時に注意
- 日本語パス対応は Go 側の ioutil でも動作するが UTF-8 を明示的に扱うこと
- `sed` は文字コード問題で使用禁止（Edit/Write ツールを使う）

## Google Drive 上でのビルド問題

Google Drive の仮想ファイルシステム (File Stream) は `node_modules` 展開時に
EBADF / EPERM が発生する。解決策としてローカルディスクにミラーしてビルドする。

- **ソース置き場**: `G:\マイドライブ\mdediter\`
- **ビルド作業場**: `C:\dev\mdediter\`
- **同期スクリプト**: リポジトリルートの `build.sh` を実行すると、
  `G:` → `C:` に src コピー → `wails build` → exe を `G:` に戻す。

```bash
# リポジトリ直下で
bash build.sh
```

`frontend/node_modules/` は G: に置かない（junction も不可）。

## ビルド時のトラブルシューティング

### Go が見つからない
```
go: command not found
```
→ PATH に `C:\Program Files\Go\bin` が含まれているか確認。
winget 直後は新規シェルが必要。

### WebView2 runtime not found
→ https://developer.microsoft.com/microsoft-edge/webview2/ から Evergreen をインストール。

### wails doctor でエラー
→ `go env` で GOPATH, GOROOT が正しいか確認。
`go install` で入る wails は `$GOPATH/bin` に配置される。
