# mdediter 仕様書

## 1. 概要

Markdown ファイルの閲覧・編集・保存を行うデスクトップアプリケーション。
Windows 上でネイティブ動作する単一実行ファイル（.exe）として配布する。

## 2. 技術スタック

| レイヤ | 採用技術 | 備考 |
|---|---|---|
| バックエンド | Go 1.22+ | ファイルI/O、設定管理 |
| フレームワーク | Wails v2 | Go と Web 技術のブリッジ |
| UIランタイム | Microsoft Edge WebView2 | Windows 10/11 標準 |
| フロントエンド | Svelte + TypeScript | 軽量・リアクティブ |
| エディタ | CodeMirror 6 | Markdown モード + シンタックスハイライト |
| Markdownパーサ | markdown-it + GFM プラグイン | 表・チェックボックス等対応 |
| コードハイライト | highlight.js | プレビュー内コードブロック |
| ビルド | Vite + wails build | 単一 exe 出力 |

## 3. 機能要件

### 3.1 必須機能（MVP）

- **ファイル操作**
  - 開く（Open）: ダイアログから .md / .markdown / .txt を選択
  - 保存（Save）: 現在開いているファイルに上書き保存
  - 名前を付けて保存（Save As）: 新規パスを指定して保存
  - 新規作成（New）: 空のタブを追加
  - 閉じる（Close Tab）: タブ単位でクローズ（未保存なら確認）

- **ライブプレビュー**
  - 左: CodeMirror6 の Markdown エディタ
  - 右: リアルタイムレンダリングされた HTML プレビュー
  - スクロール位置は独立

- **シンタックスハイライト**
  - エディタ内 Markdown 構文の色付け
  - プレビュー内コードブロックの言語別ハイライト

- **GFM 対応**
  - テーブル
  - タスクリスト（チェックボックス）
  - 取り消し線
  - 自動リンク化
  - 脚注

- **ファイルタブ**
  - 複数ファイルを同時に開く
  - タブ切り替え
  - タブ並び替え（D&D）※将来
  - 未保存インジケータ（●）

### 3.2 補助機能

- **最近開いたファイル**（Recent Files）: 最大10件、設定ファイルに永続化
- **キーボードショートカット**
  - Ctrl+O: 開く
  - Ctrl+S: 保存
  - Ctrl+Shift+S: 名前を付けて保存
  - Ctrl+N: 新規タブ
  - Ctrl+W: タブを閉じる
  - Ctrl+Tab / Ctrl+Shift+Tab: タブ切替
- **ドラッグ&ドロップ**: エディタ領域に .md をドロップして開く
- **起動時引数**: `mdediter.exe path1.md path2.md` で複数ファイルをタブとして開く（ファイル関連付け・「プログラムから開く」にも対応）
- **ダーク/ライトテーマ**: OS 設定に追従（初期はライトのみ）

### 3.3 非対応（スコープ外）

- クラウド同期
- 画像のインライン挿入補助
- WYSIWYG モード
- プラグイン機構

## 4. アーキテクチャ

```
┌─────────────────────────────────────────┐
│            mdediter.exe                  │
│  ┌─────────────┐    ┌────────────────┐  │
│  │  Go (main)  │◄──►│ WebView2 (UI)  │  │
│  │             │    │                │  │
│  │ ・FileIO    │    │ ・Svelte       │  │
│  │ ・Dialog    │    │ ・CodeMirror6  │  │
│  │ ・Config    │    │ ・markdown-it  │  │
│  └─────────────┘    └────────────────┘  │
└─────────────────────────────────────────┘
```

### 4.1 Go バックエンド構造

```
app.go              // Wails App struct
main.go             // エントリポイント
internal/
  fileio/
    reader.go       // 読み込み
    writer.go       // 保存
  config/
    config.go       // 設定読み書き (JSON)
  dialog/
    dialog.go       // OS ネイティブダイアログ
```

### 4.2 Go → JS に公開するメソッド

| メソッド | 引数 | 戻り値 | 用途 |
|---|---|---|---|
| `OpenFileDialog()` | なし | `{path, content, error}` | 開くダイアログ起動 |
| `ReadFile(path)` | path string | `{content, error}` | 指定パス読込 |
| `SaveFile(path, content)` | path, content | `{error}` | 保存 |
| `SaveFileDialog(content)` | content string | `{path, error}` | 名前を付けて保存 |
| `GetRecentFiles()` | なし | `[]string` | 履歴取得 |
| `AddRecentFile(path)` | path string | なし | 履歴追加 |
| `GetStartupFiles()` | なし | `[]string` | 起動時引数のファイルパス取得（フラグ除外・絶対パス化済） |
| `ConfirmUnsavedClose(title)` | title string | bool | 未保存タブ閉じる確認ダイアログ |

### 4.3 フロント状態（Svelte stores）

- `tabs`: `Tab[]` — 開いているタブの配列
- `activeTabId`: `string` — アクティブなタブID
- `Tab`: `{ id, path, title, content, dirty, cursor }`

## 5. ファイル・設定配置

- 実行ファイル: `build/bin/mdediter.exe`
- 設定ファイル: `%APPDATA%\mdediter\config.json`
  ```json
  {
    "recentFiles": ["C:\\...\\a.md"],
    "lastOpenedDir": "C:\\Users\\..."
  }
  ```

## 6. ビルド手順

```bash
# 開発モード（ホットリロード）
wails dev

# 本番ビルド（単一 exe 生成）
wails build -platform windows/amd64

# 出力先
build/bin/mdediter.exe
```
